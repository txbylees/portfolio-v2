The Challenge

Production failures caused by backend changes with no API test coverage

Vertex Systems had a mature SaaS product but zero formal API testing. New backend releases regularly introduced unexpected regressions that were only caught in production — often by customers. The team was aware of the risk but had never had the bandwidth to build a proper test framework.

The core challenge was not just writing tests, but designing a framework architecture that would survive team turnover, be maintainable by developers without QA experience, and integrate cleanly into their existing deployment pipeline.

The Approach

A structured Postman framework designed for longevity

I approached this as an infrastructure project as much as a testing project. The framework needed to be something the team could own and grow — not a black box that only I understood.

01

Endpoint Audit & Documentation

Catalogued all API endpoints across the platform — many of which had no documentation. Worked with backend developers to understand expected request/response contracts, authentication patterns, and error states.

02

Framework Architecture

Designed a Postman collection structure with environments, shared variables, and pre-request scripts — enabling the same tests to run against dev, staging, and production with a single environment switch.

03

Test Suite Implementation

Wrote 120+ test cases covering happy paths, error handling, authentication failures, boundary values, and data validation. Each test included assertions on status codes, response schema, and field-level data accuracy.

04

Newman CI Integration

Automated collection execution via Newman, integrated into the CI pipeline so the full API test suite ran on every deployment. HTML reports were generated automatically and linked from build outputs for easy review.

The Result

Production API incidents reduced by 4x in one quarter

Within the first quarter after rollout, production incidents caused by API regressions dropped by 75%. The team caught breaking changes in the pipeline before they reached any environment customers could see. 91% of endpoints were covered — with the remaining 9% being legacy internal tooling marked for deprecation.

A handover workshop ensured the team could maintain, extend, and debug the framework independently. The collection grew by 30% in the following quarter as the team added tests for new endpoints themselves.
